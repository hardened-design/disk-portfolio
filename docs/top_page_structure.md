# トップページ (Home) 構成案

コンセプトである「high technology low fidelity」「情報の削ぎ落とし」を体現し、予定されているギミックを自然な流れで体験できるミニマルな構成です。

## 1. ヘッダー (Header)
*   **Logo/Title**: 名前やイニシャルのプレーンなタイポグラフィ（ターミナル風フォント）。
*   **Navigation**: `About`, `Works`, `Contact` などのシンプルなアンカーリンク。
*   **Gimmick Toggle**: 【Design Guideline Toggle (Designer Mode)】のトグルスイッチ。
    *   *機能: ONにすると、サイト全体にグリッド線、ベースライン、余白の単位（px等）がオーバーレイ表示される。*

## 2. ヒーローセクション (Hero Section)
ページアクセス時に最も目を引く、サイトの「顔」となる領域。

*   **Main Visual**: 【インタラクティブ・アスキーアート】
    *   中央または背景に大きく配置された3Dアスキーアート。
    *   マウスホバーやドラッグ操作により、リアルタイムでモデルが回転・視点変更する。
*   **Tagline/Catchcopy**: 必要最小限の言葉で職能やスタンスを表現。
    *   例: `> FRONTEND DEVELOPER & UI DESIGNER_` (コマンドプロンプトやタイピングアニメーションを意識した演出)。
*   **Call to Action (CTA)**: `[ View Works ]` や `[ Scroll Down ]` などのシンプルなテキストリンク。

## 3. プロフィール (About)
*   **Content**: 簡潔な自己紹介とスキルセット。
*   **Design**: 装飾を極限まで削ぎ落としたテキストベースのレイアウト。ターミナルの標準出力（stdout）を思わせる等幅フォントでの箇条書きを取り入れ、ハッカー文化の雰囲気を出す。

## 4. 制作実績一覧 (Works)
装飾的な大きな画像を並べるのではなく、リスト形式やミニマルなカード形式で情報を整理。

*   **Public Projects**: 誰でも見られる実績。ホバー時にノイズエフェクトやアスキーアートプレビューが出るなどのlow fidelityな遊び心を入れる。
*   **Locked Projects**: 【ターミナル型パスコード入力】
    *   機密性の高い案件（NDAなど）には `[LOCKED]` のバッジを表示。
    *   クリックすると、ターミナルUIのモーダル（または全画面）が出現し `Enter Passcode:` のプロンプトが表示される。正しいコードを入力することで初めて実績詳細にアクセスできる「儀式」を演出。

## 5. フッター / コンタクト (Footer & Contact)
*   **Content**: メールアドレスやGitHubリンク、SNSリンク。
*   **Design**: 最下部にコマンド入力待ちのような点滅カーソル `_` を配置し、システムが起動し続けているような余韻を残してページを締めくくる。
