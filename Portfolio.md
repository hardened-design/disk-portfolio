# Portfolio Project Concept & Implementation Plan

## 1. サイトコンセプト：「high technology low fidelity（洗練された制約）」

グラフィックデザイナーとしての高い審美眼と、ルーツである「昔のインターネット・ナード文化」への深い愛、そして最新のWeb技術への探求心を掛け合わせたポートフォリオ。

**「情報を削ぎ落として最大限の工夫をすることへの美しさ」** をコアバリューとし、採用担当者や同業者に対して「技術への好奇心があり、自らのルーツをモダンに表現できる特異なクリエイター」としての名刺代わりとなるサイトを目指す。

*   **ビジュアル（low fidelity）**: 2chのアスキーアート、ゲームボーイのドット感、初期のWebサイトのようなソリッドなUI。あえて色数や装飾を極限まで絞る。
*   **技術・体験（high technology）**: Astroによる高速な表示、Three.jsによる滑らかな3D描画、現代的なレスポンシブデザイン。昔のカオスな文化を、UIデザイナーの「論理的思考力」で現代的に美しく再構築する。
*   **ターゲット**: 企業の採用担当者（実績と技術力の証明）、同業のクリエイター（名刺代わりのインパクト）。
*   **キーワード**: ナード、オールドインターネット、Y2K、オタク、ファッション、グラフィックデザイン、UIデザイン、論理的思考力、ENFP。

## 2. デザイン・コーディング規約（Rationale）

### CSS設計・命名規則: **Astro Scoped CSS + Minimal（機能的命名）**

*   **ルール**: クラス名は極力短く、直感的な単語（例: `.wrapper`, `.canvas`, `.title`）を使用し、Astroの標準機能であるScoped CSSによってコンポーネント単位で影響範囲を閉じる。
*   **根拠（Rationale）**:
    1.  **冗長性の排除**: Astroはビルド時にクラス名へ固有のハッシュを付与するため、BEMのような「人間が頑張って長い名前をつけて衝突を避ける」手法は二重管理になり、現代のコンポーネント設計においては冗長である。
    2.  **情報の削ぎ落とし**: コンセプトである「情報の削ぎ落とし」をコードレベルでも体現する。不要なプレフィックスを排除し、コードの純度を高める。
    3.  **コンポーネントの独立性**: ファイル単位でスタイルが完結するため、短い命名でも文脈が明確になり、可読性とメンテナンス性が向上する。
*   **技術的意図**: 「昔のインターネットのカオス（グローバルCSS）」へのリスペクトを持ちつつ、あえて「現代の技術（Astro Scoped CSS）」でそれを制御するという構造自体に、**high technology low fidelity** の精神を込める。

### スタイリングの方向性
*   **カラー**: ターミナルを彷彿とさせるモノクロ（黒/白）をベースに、Y2K/ゲームボーイ的なネオンカラー（蛍光グリーンなど）をアクセントとして極少数使用。
*   **タイポグラフィ**: 等幅フォント（Monospace）を多用し、コードエディタや古いPC画面のような規律と美しさを持たせる。

## 3. 予定機能・ギミック

「high technology low fidelity」と「情報の削ぎ落とし」を維持しつつ、実用性と遊び心を両立させる以下の機能を実装する。

### 3.1. インタラクティブ・アスキーアート
*   **内容**: マウス操作による3Dオブジェクトの回転や、視点変更。
*   **根拠**: ユーザーが直接「触れる」体験を提供することで、Three.jsによるモダンな技術力と、アスキーアートのレトロな質感をダイレクトに伝える。

### 3.2. ターミナル型パスコード入力（Locked Projects用）
*   **内容**: 公開制限のあるプロジェクトを表示する際、ターミナルUIが出現し、簡易的なパスコード入力を求める。
*   **根拠**: ナード/ハッカー文化へのオマージュ。単なる入力フォームではなく、世界観に合わせた「儀式」として提供することで、機密性の高い実績への関心を高める。

### 3.3. デザイン・ガイドライン・トグル（Designer Mode）
*   **内容**: サイト全体のグリッド、ベースライン、余白の単位（px等）を可視化するオーバーレイ表示の切り替え機能。
*   **根拠**: グラフィックデザイナーとしての「論理的思考力」と「UI設計の緻密さ」を視覚的に証明する機能。審美眼の裏側にある設計意図を、同業者や採用担当者にアピールする。

### フェーズ 4: 検証と調整
